# AI 漫剧平台 MVP 产品方案

## 1. MVP 产品定位

面向独立创作者、小团队、AI 内容运营者的“AI 漫剧分镜生产工作台”。第一版目标不是直接生成商业级完整短剧，而是把从故事到可预览分镜的流程做稳定：结构化剧本、角色设定、镜头分镜、镜头图生成、素材管理、任务追踪、可导出。

一句话定位：

> 输入故事，管理角色，生成分镜和镜头图，用可替换 AI Provider 逐步扩展到视频。

## 2. 核心功能清单

第一版必须做：

- 项目管理：创建、编辑、归档项目。
- 剧本输入：粘贴故事梗概/短剧剧本，保存为项目原文。
- AI 剧本结构化：从原文生成角色、场景、分镜草案。
- 角色管理：角色名、描述、性格、外观、参考图、音色占位字段。
- 分镜管理：镜头序号、场景、角色、旁白/对白、画面描述、镜头类型、时长估算。
- 镜头图生成任务：每个分镜可生成一张图，保存任务状态和产物。
- Mock Provider：无模型也能返回占位图、假任务进度、模拟错误。
- ComfyUI Provider：通过配置 ComfyUI 地址和 workflow template 调用真实图片生成。
- 素材库：上传/查看/复用图片、生成结果、角色参考图。
- 任务中心：查看进行中、成功、失败的生成任务，可重试。
- 导出：导出项目 JSON、Markdown 分镜表、镜头图 ZIP。

第二阶段可做：

- 图生视频 Provider。
- TTS 配音。
- 单镜头视频合成。
- 整集拼接。
- 分镜批量生成队列。
- 简单画布视图。
- 多用户与权限。

## 3. 暂不做的功能

- 不做完整无限画布。
- 不做专业视频剪辑时间线。
- 不做社媒发布。
- 不做模型训练、LoRA 训练、角色微调。
- 不做复杂计费系统。
- 不做多租户企业权限。
- 不做在线 ComfyUI 节点编辑器。
- 不做自动投流、数据分析和内容分发。
- 不内置任何第三方大模型密钥。

## 4. 用户流程

### 4.1 首次使用

1. 打开平台。
2. 在设置页选择 `Mock Provider` 或配置 `ComfyUI Provider`。
3. 创建第一个项目。
4. 输入故事梗概或短剧文本。
5. 点击“生成结构化分镜”。
6. 系统生成角色、场景、分镜草案。
7. 用户编辑角色外观和每个镜头描述。
8. 为镜头生成图片。
9. 查看任务中心和素材库。
10. 导出分镜方案。

### 4.2 常规创作

1. 从项目列表进入项目。
2. 在“剧本”页修改故事或单集内容。
3. 在“角色”页维护角色参考。
4. 在“分镜”页编辑镜头和生成提示词。
5. 在“生成”面板选择 Provider、尺寸、风格、seed。
6. 后台生成任务并保存结果。
7. 用户选择满意版本作为镜头主图。
8. 导出供后续剪辑或视频生成使用。

## 5. 前端页面清单

| 页面 | 路由建议 | 说明 |
| --- | --- | --- |
| 项目列表 | `/projects` | 项目卡片、状态、最后更新时间、创建按钮 |
| 项目概览 | `/projects/:id` | 项目摘要、角色数、分镜数、任务状态 |
| 剧本页 | `/projects/:id/script` | 原文输入、结构化生成、版本记录 |
| 角色页 | `/projects/:id/characters` | 角色表、参考图、外观描述 |
| 分镜页 | `/projects/:id/storyboard` | 镜头表格/卡片、对白、画面描述、生成按钮 |
| 镜头详情 | `/projects/:id/shots/:shotId` | 单镜头 prompt、生成结果、版本选择 |
| 素材库 | `/projects/:id/assets` | 图片上传、生成结果、角色引用 |
| 任务中心 | `/tasks` | 全局任务列表、状态、重试 |
| Provider 设置 | `/settings/providers` | Mock/ComfyUI 配置、连通性测试 |
| 工作流模板 | `/settings/workflows` | ComfyUI workflow JSON、输入映射 |

## 6. 后端模块清单

| 模块 | 责任 |
| --- | --- |
| `projects` | 项目 CRUD、归档、统计 |
| `scripts` | 原文保存、结构化结果版本 |
| `characters` | 角色 CRUD、参考图绑定 |
| `scenes` | 场景 CRUD、场景描述 |
| `shots` | 分镜 CRUD、排序、镜头状态 |
| `assets` | 上传、生成产物、文件元数据 |
| `tasks` | 生成任务创建、状态流转、重试 |
| `providers` | Mock、ComfyUI、未来云模型 Provider 抽象 |
| `workflowTemplates` | ComfyUI 工作流模板和变量映射 |
| `storage` | 本地文件存储，未来扩展 S3/R2/OSS |
| `exports` | Markdown/JSON/ZIP 导出 |
| `settings` | 模型配置、系统配置、密钥引用 |

## 7. MVP 成功标准

- 不配置任何真实模型时，Mock Provider 能完整跑通项目创建、分镜生成、镜头图生成、导出。
- 配置可用 ComfyUI 后，至少一个镜头能生成真实图片并入库。
- 所有生成任务都有可追踪状态：`queued`、`running`、`succeeded`、`failed`、`cancelled`。
- 角色、分镜、素材之间关系清晰，可导出给人类继续制作。
- 后续接入视频/TTS 时不需要重写项目/分镜/任务/素材核心模型。
